# Product-Focused PowerPoint Design System

## Overview

This document defines the visual and structural design system for a product-focused PowerPoint presentation. The goal is to create a clean, modern, and non-distracting theme that emphasizes product thinking, user value, and clear storytelling.

The design should feel polished and professional, similar to internal product strategy decks used at leading technology companies.

---

## Core Design Principles

- Keep visuals calm, minimal, and intentional
- Prioritize clarity, readability, and flow
- Emphasize storytelling and product outcomes
- Use whitespace to guide attention
- Avoid unnecessary decoration or visual noise

---

## Color System

### Base Colors

- **Background:** #F7F9FB (off-white / light gray)
- **Primary Text:** #1A1A1A (near-black)
- **Secondary Text:** #6B7280 (muted gray)

### Accent Colors (Use Sparingly)

- **Primary Accent:** #4A90E2 (muted blue)
- **Secondary Accent (optional):** #5FA8A0 (soft teal)
- **Highlight (rare use):** #E6A700 (subtle amber)

### Rules

- Avoid neon or highly saturated colors
- Avoid large color blocks behind text
- Use only one accent color per slide
- Maintain a soft, low-contrast visual tone

---

## Typography

- Use clean sans-serif fonts:
  - Preferred: Inter, Segoe UI, Calibri
- **Titles:** Semi-bold (not heavy or extra bold)
- **Body Text:** Regular weight
- Avoid all caps (except small labels)
- Use size and spacing for hierarchy, not color

---

## Layout Guidelines

- Use consistent margins and spacing
- Prefer left-aligned content
- Use simple grid structures
- One key idea per slide
- Maintain clear hierarchy:
  - Headline → Supporting Points → Optional Detail

---

## Slide Structure (Product-Oriented)

Each slide should communicate a clear product insight or decision.

### Recommended Flow:

- Problem
- Insight
- Solution
- Impact

### Headline Style:

- Write headlines as insights, not labels

**Good Example:**

> Users Drop Off During Onboarding

**Bad Example:**

> Onboarding Flow

---

## Visual Elements

### Diagrams (Preferred)

- User journeys
- Product flows
- Roadmaps (Now / Next / Later)
- Feedback loops

### Style Rules

- Keep diagrams simple and functional
- Use thin lines and minimal shapes
- Avoid detailed UI mockups unless necessary
- Avoid decorative icons

---

## Charts & Data Visualization

- Focus on trends and insights
- Keep styling minimal
- Use muted tones
- Highlight only key data points using accent color

---

## Agile + Product Integration

- Reflect iterative delivery:
  - MVP → Iteration → Scale
  - Now / Next / Later
- Emphasize outcomes over outputs
- Show prioritization and trade-offs clearly
- Support product storytelling across slides

---

## Content Guidelines

- Use short, meaningful bullet points
- Avoid dense paragraphs
- Highlight key insights subtly
- Focus on user value and business impact

---

## What to Avoid

- No dark mode slides
- No bold or high-contrast color palettes
- No heavy gradients or shadows
- No oversized headers dominating slides
- No cluttered or overly technical visuals (unless required)
- No more than one accent color per slide

---

## Tone & Feel

The presentation should feel:

- Thoughtful and user-focused
- Calm and professional
- Clean and structured
- Confident but not flashy

It should resemble work produced by a strong product manager or product-minded engineer presenting to leadership.

---

## Consistency Rule

All slides must follow this design system consistently:

- Same spacing
- Same typography
- Same color usage
- Same layout logic

Consistency is more important than creativity.
